## 1. 보스 오리지널 이름 — **the Bracken Warden**

The Old Threshold 는 한때 무엇인가에 의해 지켜졌다. Verdant Ruin 의 사람들이 세계에서 옆으로 비켜선 후 오랜 시간이 지난 후에도, 그들이 마지막 방에 둔 것은 자기 위치를 지켰다. 이끼가 그 어깨 위에 자랐고, 고사리 잎이 그 관절에 뿌리내렸고, 돌 이음새가 돌 이음새 안으로 무너졌다 — 하지만 **Warden** 은 머물렀다. 그것은 Reed 를 쫓지 않는다; 그를 미워하지 않는다; 방이 그것을 깨웠기에 그저 머리를 들어 올린다. **위협-읽기: 방의 일부가 일어남.**

**만들어낸-이름 어원.**
- *Bracken* — 수세기에 걸쳐 Warden 안으로 자라난 야생, 넓은-잎 고사리. 본 작품에 오리지널; 어떤 저작권 있는 생명체 이름도 아님.
- *Warden* — 일반 영어 단어 ("장소의 가디언"); 우리가 아는 어떤 IP 에서도 고유 명사가 아님.
- 결합하면, **Bracken Warden** 은 영어와 한국어 모두에서 특정 Wonder Boy 보스와의 음성적 중복 없이 읽힘 (v0.50 캐스트 — Mossplodder, Hummerwing, dawn-husk, Reed Bramblestep — 과 `docs/story/world.md` 로어에 대해 검증). 한국어 표기: **브랙큰 워든** (음역; 이름은 `CLAUDE.md` 바이링구얼 정책 따라 그대로 유지).

**왜 이 이름이고 다른 이름이 아닌지.**
- 그것은 **뿌리내림** — 보스는 이끼와 돌, 금속적이거나 공격적이지 않다.
- 그것은 직책-제목 없이 **묘사적** — "Warden" 은 그것이 하는 일, "Bracken" 은 그것이 보이는 모습.
- 그것은 **생물 클래스가 아님** — 플레이어는 나중에 더 많은 "Brackens" 와 싸우지 않음. 이것은 *그* Bracken Warden. 단수.
- 그것은 **자세에서 부드러움** — `docs/story/world.md` ("gentle, not grim") 와 일관. The Warden 은 악의적이지 않다; 그것은 *집요*하다.

## 2. 실루엣 의도

아레나의 뒷벽에 있는 **무릎 꿇은 인간형 거상**. 풀로 일어났을 때 ~3 타일 폭 × 4 타일 높이 (브리프의 "측면당 3-4 타일" 범위의 더 큰 끝 — 에셋 크기는 §10 참고). 휴식 시 (Reed 진입 전 초기 상태): Warden 은 앞으로 웅크리고 앉아, **한 무릎은 바닥에, 두 팔뚝은 굽힌 무릎에 놓고, 머리는 숙임**. 실루엣은 **이끼-덮인 cairn 모양**으로 읽힘 — 둥근 형태의 스택, 가장 위는 숙인 머리. 캐주얼 시청자의 눈은 그것이 움직이기 전까지 방의 지형 일부 (조각상, 이끼-둔덕) 로 읽어야 함.

**일어선 상태** (`idle` 후 어느 상태든): 머리가 올라가고, 가슴이 열리고, 두 팔이 펼침. 가슴이 **새겨진 dawn-amber 시질**을 드러냄 — 따뜻하게 펄스하는 수직 슬릿 모양. 이것은 Warden 의 살아 있는 핵심; 또한 **시각적 텔레그래프 허브** — 공격으로의 모든 와인드업이 시질을 플레어.

**자세는 "포식자"가 아니라 "지킴"으로 읽혀야 함.** Warden 은 낮게 웅크리거나 감지 않는다; 일어서서 기다린다. 아레나는 그것의 것. Reed 가 들어온 자.

**재료 읽기:**
- 어깨, 팔꿈치, 무릎의 이끼 사이로 보이는 돌 이음새 — 고사리가 덮을 수 없었던 곳.
- 엉덩이, 손가락 사이, 척추를 따라 있는 고사리 잎 (넓은 부채-잎 컬).
- 다른 모든 곳에 이끼 — 두꺼운, 층층의, 가끔씩 늘어진.
- 새겨진-돌 얼굴: 눈이 있을 곳에 수평 슬릿을 제외하면 특징 없음. 슬릿은 Warden 이 완전 `windup` 또는 `attack` 일 때만 가슴 시질과 같은 dawn-amber 로 빛남 — `idle` 에서 슬릿은 어둡다.

**스케일 읽기.** Reed 는 1 타일 높이 (36 × 48 스프라이트). Bracken Warden 은 일어났을 때 ~3 타일 폭 × 4 타일 높이, 무릎 꿇었을 때 ~3 타일 폭 × 3 타일 높이. 크기 비율은 ~1:4 — 아레나의 12 × 11 타일 직사각형을 채우지 않고 압도할 만큼 큼.

## 3. 움직임 FSM

상태: `idle`, `windup`, `attack`, `recover`, `hurt`, `dead`.

```
        ┌───────── idle ──────────┐
        │ (무릎 꿇음, 머리 숙임)   │
        ▼                         │
   Reed 가 아레나에 진입? 또는      │
   recover 타이머 만료?           │
        │ 예                       │
        ▼                         │
   ┌─────────┐  ~45 프레임   ┌─────────┐  쇼크웨이브 스폰  ┌──────────┐
   │ windup  │ ────────────► │ attack  │ ──────────────────►│ recover  │
   │ (시질이  │               │ (팔     │                     │ (시질    │
   │  플레어)│               │  슬램)  │                     │  페이드) │
   └─────────┘               └─────────┘                     └────┬─────┘
                                                                   │ (~90 프레임)
                                                                   ▼
                                                                  idle

   어떤 non-dead 상태든 ── 도끼 적중 ──► hurt ── (~10 프레임) ──► 마지막 상태
   어떤 상태든 ── 7번째 도끼 적중 ───────► dead   (종결)
```

Dev 를 위한 노트:
- **`idle` 은 휴식 상태**, 배회 상태가 아님. Warden 은 `idle` 에서 **움직이지 않음**. 무릎 꿇은 자세 유지. `idle` 의 첫 진입 (초기 파이트-전 휴식) 은 특별함: Reed 가 아레나 트리거 컬럼을 건널 때까지 무한정 `idle` 유지. 모든 후속 `idle` 재진입 (after `recover`) 은 `idleFrames` ≈ 60 프레임 동안 유지한 후 `windup` 재진입.
- **`windup` 은 텔레그래프.** Warden 이 머리를 들고, 그 다음 한 팔을 머리 위로 든다. 가슴 시질이 전체 `windup` 윈도우에 걸쳐 어두운 호박색에서 밝은 호박색으로 플레어. 이것은 플레이어의 반응 윈도우 — 점프하거나, 지나서 스프린트하거나, 도끼 던지기를 포지셔닝할 ~45 프레임 (~0.75 초) 이 있음.
- **`attack` 은 슬램.** 들어 올린 팔이 바닥으로 강하게 내려옴; 접촉-프레임에서, **moss-pulse 쇼크웨이브**가 Warden 의 발에 스폰되고 바닥을 따라 Reed 쪽으로 왼쪽으로 이동 (공격 패턴은 §4 참고).
- **`recover` 는 쿨다운.** 팔이 바닥에 늘어진 채 유지; 시질이 전체 `recover` 윈도우에 걸쳐 밝은 호박색에서 어두운 호박색으로 페이드. Warden 은 `recover` 동안 도끼 적중에 가장 취약 (Design 이 이 상태에서 가슴 시질이 가시적으로 노출되도록 묘사). ~90 프레임.
- **`hurt` 는 짧은 비틀거림.** ~10 프레임 동안 Warden 의 실루엣이 플래시 (1-프레임 모두-밝게 스왑, v0.50 Mossplodder 적중-플래시 따라, 그 다음 9 프레임 유지). Warden 의 현재 상태-위치는 보존됨 — windup 타이머가 재시작되지 않고 그저 hurt 윈도우 동안 일시정지. 플레이어가 몇 적중을 착지하든 공격 사이클을 예측 가능한 비트에 유지.
- **`dead` 는 종결.** 사망 애니메이션 (§5) 재생 후 Area Cleared 오버레이 흐름 트리거 (`phase3-area1-expansion.md` §8). Warden 은 사망 anim 동안 디스폰되지 않음; 시신은 페이드-투-블랙까지 보임.

**상태-트랜지션 트리거 — 형식.**

| From → To           | 트리거                                     |
|---------------------|---------------------------------------------|
| pre-`idle` → `idle` | 보스 아레나 스폰 — 무릎 꿇은 자세 유지.     |
| `idle` → `windup`   | 파이트-전: Reed 가 `bossArenaTriggerCol` 건너기. Post-recover: `idleFrames` 경과. |
| `windup` → `attack` | `windupFrames` 경과 (~45 프레임).        |
| `attack` → `recover`| 접촉-프레임에 쇼크웨이브 스폰 (`attack` 의 ~12 프레임 in). 팔-바닥-슬럼프 ~6 프레임 후, `recover` 로 트랜지션. |
| `recover` → `idle`  | `recoverFrames` 경과 (~90 프레임).       |
| 어떤 (dead 아님) → `hurt` | 도끼가 Warden 의 히트박스에 적중. Hurt 는 짧은 오버레이; 상태 머신이 이전 상태로 복귀. |
| 어떤 (dead 아님) → `dead` | 도끼 적중 후 `hp` 가 0 에 도달.      |

## 4. 공격 패턴 — the **moss-pulse 쇼크웨이브**

ONE 주 공격. `windup` 팔-올리기 + 가슴-시질 플레어로 텔레그래프. `attack` 시작 시 단일 접촉-프레임에 스폰.

**설명.** 슬램 시, Warden 발의 지면이 이끼-와-돌 조각의 파동으로 **바깥쪽으로 펄스**. 파동은 바닥을 따라 **왼쪽으로 (Reed 의 스폰 쪽)** 일정 속도로 이동. 그것은 **단일 지면-허그 엔티티** — Reed 는 그 위를 점프하거나, 지나서 대시 (스프린트 = X-홀드) 하거나, 도끼로 막을 수 있음 (도끼가 펄스를 흡수하고 디스폰; 둘 다 같은 프레임에 죽음). 파동은 도달하면 **아레나의 왼쪽 벽**에서 디스폰.

**왜 이 공격, 다른 것이 아닌지.**
- **본 프로젝트에 오리지널** — 불-스핏 (Drancon) 아님, 파이어볼 (Hottentots) 아님, 스우핑 다이브 (날아다니는 보스 없음) 아님. 지면-펄스 + 점프-오버는 보편적인 플랫포머 비트, 저작권 있는 움직임 아님.
- **Reed 의 기존 키트에 작동.** 점프 (Z), 스프린트 (X-홀드), 도끼 (X-탭). 셋 모두 파동에 대응. 플레이어는 이미 아는 것을 사용.
- **분명히 텔레그래프.** ~45 프레임 `windup` (~0.75 초) 은 경험 있는 플레이어에게 충분하고 새로운 플레이어에게 정확히 충분.
- **공간적으로 읽기 쉬움.** 파동은 바닥을 허그, 한 방향으로 이동, Reed 의 고도를 추적하지 않음. 접촉-프레임에 공중에 있는 Reed 는 **안전**; 바닥에 있는 Reed 는 반응해야 함.
- **플레이어가 반격할 수 있음.** `recover` 동안 (~90 프레임), Warden 의 가슴 시질이 노출. Reed 는 공격-사이클당 대략 1.5 초의 시간이 있어 시질에 도끼를 던질 수 있음.

**파동 속성.**

| 속성                       | 제안값                    | 비고                                             |
|---------------------------|--------------------------|-------------------------------------------------|
| `wave.heightTiles`        | 1                        | 단일-타일 높이. Reed 가 그 위로 점프.            |
| `wave.widthTiles`         | 1                        | 단일-타일 폭 전방 면.                            |
| `wave.vx`                 | -3.5 px/frame            | 왼쪽으로 이동. Reed 의 걷기와 같은 속도 — *플레이어는 스프린트해서 파동을 앞지를 수 있음*. (스프린트 1.4× = 4.9 px/frame > 3.5.) |
| `wave.contactDmg`         | kill                     | Reed 접촉 시 1-hit-kill (v0.25.2 규칙 따라).     |
| `wave.lifetimeFrames`     | 240                      | 하드 캡 (~4 초). 먼저 왼쪽 벽 접촉 시 디스폰.    |
| `wave.hatchetInteraction` | mutual-despawn           | 도끼가 파동 죽임; 파동이 도끼 죽임.              |

**스폰 시퀀스 (프레임-별).**

| `attack` 시작 후 프레임      | 이벤트                                                                |
|---------------------------|----------------------------------------------------------------------|
| 0                         | `attack` 상태 시작. 시질 최대 밝기. 팔 최고점.        |
| 0-11                      | 팔이 아래로 휘두름 (3 애니메이션 프레임, 각 4 프레임).               |
| 12 (접촉-프레임)            | **쇼크웨이브 엔티티가 Warden 발에 스폰, vx = -3.5.** 시질이 한 단계 어두워짐. |
| 12-17                     | 팔이 바닥에 쉼 (6 프레임 유지).                          |
| 18                        | `recover` 상태로 트랜지션.                                       |

**파동 시각.** 선두 면 (1-타일 높이) 에 이끼-와-돌 입자의 짧은 위쪽 폭발, 선두 면 뒤로 ~10 px 에 걸쳐 페이드되는 후행 지면-블러. 색상: 이끼 입자에 `mosaic-cool`, 파동 베이스의 따뜻한 글로우에 `dawn-channel-amber`, 후행 블러에 `velvet-shadow`. 파동은 v0.50 의 어떤 엔티티와도 시각적으로 분명히 다르게 읽혀야 함 — v0.50 에 수평으로 지면을 허그하는 해저드나 발사체가 없음.

**엣지 케이스: 스폰 시 파동 안의 Reed.** Reed 가 파동 스폰 시 Warden 의 발 바닥에 있으면 (불가능하지 않지만 가능 — Reed 가 windup 안으로 직접 걸어 들어가 점프하지 않음), Reed 는 접촉-프레임에 죽음. 쇼크웨이브-그레이스-프레임 없음; 이것은 v0.50 즉시-킬 자세와 일관.

## 5. 적중 반응

**도끼 적중이 Warden 을 짧게 비틀거리게 함.** 도끼가 Warden 의 히트박스에 적중할 때:

- 현재 상태의 타이머가 `hurtFrames` (≈ 10 프레임) 동안 **일시정지**.
- Warden 의 실루엣이 플래시: 프레임 1 = 모두-밝은 팔레트 스왑 (1-프레임; v0.50 적중-플래시 컨벤션 매칭), 프레임 2-10 = 시질이 잠시 페이드된 이전 상태의 마지막 가시 프레임 유지.
- HP 가 1 감소.
- 도끼는 **접촉 시 디스폰**, 어떤 엔티티에 대한 어떤 Phase 2 도끼 적중과도 같음 (바운스 없음; `phase2-cast-revision.md` §6 따라).
- **넉백: 없음.** Warden 은 밀려나지 않음. 무릎 꿇은 거상이다; 그 자리에 머문다.

**HP: 6 적중.** 제안 시작값 (Dev 튜닝). 브리프의 5-7 적중 범위 내. 6 적중에 2-도끼 온스크린 캡과 12-프레임 공격 쿨다운으로, 플레이어는 `recover` 윈도우당 1-2 적중 (~90 프레임) 을 착지할 수 있음. 파이트는 대략:

- 공격 사이클 = 45 (windup) + 18 (attack-with-spawn) + 90 (recover) + 60 (idle) ≈ 213 프레임 (~3.55 초) 루프당.
- 플레이어가 사이클당 1 깔끔 적중 = 6 사이클로 킬 = ~21 초, 회피 시간 포함.
- 플레이어가 사이클당 2 깔끔 적중 = 3 사이클로 킬 = ~10 초, 완벽한 던지기 타이밍 필요.

**히트박스.** Warden 의 도끼-취약 히트박스는 **가슴 시질**에 중심 (실루엣 중앙의 대략 1-타일 높이 × 1-타일 폭 영역). 시질을 빗나가 이끼/고사리에 적중하는 도끼는 무해하게 디스폰. 이것은 던지기에 스킬을 둠: `recover` 동안 가슴 높이의 깔끔한 던지기가 착지; 무릎 꿇은 Warden 의 발에 대한 낮은-호 던지기는 않음.

**마지막 적중에서 (HP → 0).** Warden 이 `dead` 진입. 사망 애니메이션 §6 재생. 보스 아레나 흐름이 `phase3-area1-expansion.md` §8 따라 진행.

## 6. 색상-무드 키워드

Design 용 (hex 는 `docs/design/palette-phase2.md` 컨벤션 따라 선택; 본 브리프는 무드 앵커만 명세):

- `bracken-frond green` — 넓은-잎 고사리 녹색. Stage 1 의 `moss-green` 의 **약간 더 차갑고, 더 채도 높은** 사촌 — 고사리가 길 이끼보다 더 깊이 자랐다. Hex 자매 가족: `4a7c3a` / `3e6a3a`-인접.

- `cairn-mantle cool` — 휴식 시 이끼-덮인 외부 실루엣. `bracken-frond green` 의 더 깊은 사촌; 이끼가 수년에 걸쳐 고사리 위에 층을 쌓았다.
- `sigil-amber peak` — `attack` 동안 완전 플레어된 가슴 시질. 전체 보스 팔레트에서 가장 밝은 호박색이어야 함 — `#f8d878` 재사용 (Sapling amber-bright, 은퇴했지만 사용 가능; `palette-phase2.md` 공유-hex 정책 따라).

**크로스-팔레트 일관성 규칙 (`palette-phase2.md` §"Cross-sprite consistency rules — Phase 2" 따라):**

## 8. 아레나 레이아웃

**아레나 치수.** ~12 타일 폭 × 11 타일 높이. 뷰포트의 16 × 12 타일 공간보다 약간 적어 HUD 스트립이 화면 상단에 가시적으로 유지.

```
                          (HUD 스트립 — 상단 1 타일이 lives/vitality 용으로 예약)
        ┌───────────────────────────────────────────────────────────┐ (행 0 — 아레나 밖, HUD 만)
        ▕                                                            ▕
        ▕   (닫힌 지붕 — 유적 챔버의 돔형 천장 — 시각)                  ▕
        ▕                                                            ▕
        ▕                                                            ▕
        ▕                                                            ▕
        ▕                                                            ▕
        ▕         (열린 아레나 내부 — 빈 바닥 + 벽)                    ▕
        ▕                                                            ▕
        ▕                                                            ▕
        ▕    R (Reed 가 여기서 진입)                            W →   ▕  W = Warden, 무릎 꿇음, 왼쪽 향함
        ▕    →                                                       ▕
        █████████████████████████████████████████████████████████████  (행 11 — 솔리드 아레나 바닥)
        col 0                                                     col 11
                                                                       (아레나 오른쪽 벽은 솔리드; Warden 이 그 너머로 움직일 수 없음; Reed 가 그 너머로 탈출할 수 없음)
```

**구체적 사항.**
- **아레나 내부:** cols 0-11 (12 타일 폭), 아레나 내 rows 0-10 (11 행 높이). 바닥 행 = 행 10 (1 타일 높이).
- **벽:** 아레나의 왼쪽 벽은 **카메라-잠금 경계**, 솔리드 타일 그 자체는 아님 — Reed 가 앞방에서 연속적으로 진입; 카메라가 Reed 가 완전히 안에 있을 때 잠금. 아레나의 오른쪽 벽은 **솔리드**: col 11 의 1-타일-두께 컬럼, 전체 11 행 높이.
- **천장:** 데코레이션 (유적의 돔형 천장으로 그려짐). Reed 가 점프로 닿을 수 없음 (점프 정점 ≈ 2.3 타일 ≪ 11 타일).
- **Reed 스폰:** col 1, row 9 (바닥 위 한 타일, 왼쪽 가장자리에서 한 타일 안). Reed 가 오른쪽을 향해 진입.
- **Warden 스폰 (무릎 자세):** col 9, row 7 (오른쪽 벽에서 한 타일 안, 무릎 자세 실루엣이 ~행 7-10 을 차지). 왼쪽 향함.
- **카메라 잠금 위치:** 카메라가 아레나의 왼쪽 가장자리를 뷰포트 col 0 에 두고 앵커. 뷰포트는 16 타일 폭; 아레나는 12 타일; 오른쪽으로 남은 4 뷰포트 컬럼은 앞방의 연속 또는 닫힌 천장을 오른쪽으로 확장해 보여줌, design-lead 의 결정.

**왜 이 레이아웃.**
- 12 폭이 Reed 에게 측면당 한 타일 여백으로 ~10 타일의 달리는 공간 제공 — 스프린트로 쇼크웨이브를 앞지를 수 있을 만큼 충분, 무시할 수 있을 만큼 많지 않음.
- 11 높이가 Reed 가 Stages 1-3 에서 훈련해온 수직성 보존 (그의 점프 정점이 중요; 쇼크웨이브는 점프-오버 가능).
- 솔리드 오른쪽 벽 = Warden 이 후퇴할 수 없음; 파이트가 한 방식으로 끝남.
- 아레나 내 바닥 갭 없음 = Warden 의 쇼크웨이브가 구멍에 빠지지 않고 바닥을 따라 깨끗하게 이동 가능. 공간적으로 깨끗.

## 9. 파이트 동안 카메라 동작

**파이트-전.** 표준 v0.50.1 카메라 lerp: 플레이어 왼쪽에서 ~1/3, 스테이지 가장자리로 클램프. Reed 가 Stage 4 Round 4-4 앞방을 통과해 걷는 동안 카메라가 정상 추적.

**아레나 진입 시 (Reed 가 Round 4-4 의 col 32 건넘).** 카메라가 아레나의 왼쪽 가장자리를 뷰포트 col 0 에 맞춰 **잠금**. 카메라는 파이트 동안 스크롤하지 않음. Reed 의 자유 움직임 보존; 카메라만 고정.

**파이트 동안.** 카메라 잠금. 스크롤 없음.

**보스 사망 시 (HP → 0).** 카메라는 사망 애니메이션 (~60 프레임) 동안 잠금 유지. 사망 anim 후, ~60-프레임 축하 일시정지 (Reed 가 움직일 수 있지만 할 일이 없음). 그 다음 Area Cleared 오버레이 페이드 발화 (`phase3-area1-expansion.md` §8).

**아레나 내 Reed 사망 시.** dying-frames 시작 시 카메라가 **즉시 잠금 해제** (사망 애니메이션이 정상 스크롤 컨텍스트에서 재생되도록). `mile_16` (Round 4-4 col 3) 에서 리스폰 후, 카메라가 Reed 를 앞방을 통해 정상적으로 추적; col 32 재-건너기 시, 카메라 재-잠금.

**왜 Reed 사망 시 잠금 해제하고 보스 스폰-재시도에서는 아닌가?** 두 이유: (1) 보스가 스폰-후 무릎 자세로 돌아가는 것은 카메라 스크롤이 필요 없지만, Reed 가 아레나로 다시 걷는 것은 필요; (2) 플레이어의 사망 애니메이션은 스테이지의 다른 어디서나 사망과 같은 화면 처리를 받을 자격이 있음 — 사망 동안 잠긴 카메라는 Reed 의 사망을 스테이지 사망과 다르게 느끼게 만들어 혼란.

## 12. 승리 조건

**보스 HP 가 0 에 도달 → 사망 애니메이션 재생 → 축하 일시정지 → Area Cleared 오버레이.**

상세 시퀀스 (`phase3-area1-expansion.md` §8 따라):

1. `bracken.hp = 1` 에 도끼 접촉이 `bracken.hp → 0`. `dead` 로 트랜지션.
2. `dead` 애니메이션 재생 (~60 프레임, §11.1 `deathFrames` 따라).
3. 축하 일시정지 (~60 프레임, §11.1 `celebrationFrames` 따라). Reed 가 움직일 수 있지만 공격 불가 (유일한 공격 가능한 타겟이 없음; 카메라가 여전히 잠금).
4. ~60 프레임에 걸친 페이드-투-블랙.
5. **Area 1 Cleared 오버레이** ~5 초 유지 (또는 입력 시까지):
   ```
   Area 1 — The Mossline Path
   Cleared.

   The threshold opens. Reed walks on.

   Area 2 ahead in v0.85.
   ```
   KO 파일의 바이링구얼 페어. "v0.85" 라인은 릴리스-정직 카피 — 오픈 질문 §15 참고.
6. 종결 비트: 타이틀 화면, 또는 오버레이를 무한정 유지 (dev-lead 의 결정; v0.50 cairn-클리어 의례와 일관).

**런-베스트 캡처.** v0.75 범위 밖. Release-master 가 미래 리더보드 기능을 원하면, 이 비트에서 캡처할 값: 남은 `state.lives`, Area 1 스폰 이후 총 경과 프레임, Warden 을 죽이는 데 받은 도끼-적중. 그 어느 것도 v0.75 오버레이 카피에 영향을 주지 않음.

## 13. 패배 조건

Reed 가 보스 아레나 안에서 지는 두 가지 방법. 둘 다 표준 v0.50.2 dying + 리스폰 흐름; 새 것 없음.

**1. 보스 접촉 (`bracken` 가 `dead` 를 제외한 어떤 상태든).** Reed 가 Warden 의 몸 히트박스에 걸어 들어감 = 표준 접촉-데미지 경로를 통한 `state.killHero(player)`. 영웅 접촉 히트박스 vs. Warden 몸 히트박스; v0.25.2 규칙 따라, 어떤 non-dead 적 상태도 1-hit-kill.

**2. Moss-pulse 쇼크웨이브 접촉.** Reed 가 활성 쇼크웨이브 엔티티에 닿음 = `state.killHero(player)`. `mosspulse.contactDmg = kill` 따라. 공중에서 (쇼크웨이브의 1-타일 실루엣 위) Reed 는 **안전**; Reed 가 죽으려면 바닥 고도에 있어야 함.

**리스폰 흐름** (`phase3-area1-expansion.md` §8 + v0.50.2 dying FSM 따라):

- `state.beginDying(player)` → 45-프레임 dying 타이머 + 넉백 + 사망 anim.
- `state.loseLife()` → `state.lives -= 1` → `mile_16` (Round 4-4 col 3) 에서 `pl.armed` 보존된 채 리스폰.
- **보스 리셋:** Warden 이 스폰 위치의 `idle` 로 돌아감, 전체 HP, 모든 타이머 청소. 카메라 잠금 해제 (Reed 가 앞방에 돌아옴).
- **재진입:** Reed 가 앞방을 다시 걷고, col 32 를 재-건너고, 카메라가 재-잠금, Warden 이 처음부터 `idle → windup` 사이클 시작.

**GAME OVER 경로.** 보스 파이트 안에서 `state.lives` 가 0 에 도달하면, v0.50.2 GAME OVER 흐름 실행. Continue 후:
- 라이프가 3 으로 리필.
- Vitality 가 최대로 채워짐.
- StageManager 가 **Stage 1 스폰에서 전체 Area 를 다시 빌드**, 무장 해제. `phase3-area1-expansion.md` §2 따라 — 전체 Area 리셋.
- 보스는 Continue 간에 받은 데미지의 기억이 없음.

**Stage 4 재시작 단축키 없음.** 보스에서 라이프가 다 떨어진 플레이어는 "Stage 4 스폰에서 재시작" 자비를 받지 **않는다**. 그들은 Stage 1 에서 재시작. 이것은 Phase 2 의 정신과 일관 — 런이 여정; 보스가 보상 비트. Release-master 는 접근성을 위해 이것을 부드럽게 할 수도 — §15 에 플래그.

## 14. 스모크 체크 (v0.75 쿼타일 게이트 — 보스-특정)

area-expansion 브리프 §13.6 + 본 브리프 따라:

1. **보스 진입.** Reed 가 Round 4-4 col 32 를 건넘. 카메라가 뷰포트 col 0 = Round 4-4 col 32 에 잠금. Warden 이 `idle → windup` 트랜지션 재생; 시질 플레어.
2. **텔레그래프 읽기.** 45 프레임에 걸쳐 재생되는 `windup` 애니메이션 봄; 마지막 프레임까지 팔이 머리 위로 올라가는지 검증. 시질 최대 밝기.
3. **쇼크웨이브 스폰.** `attack` 의 프레임 12 에서, 쇼크웨이브 엔티티가 Warden 의 발 (Round 4-4 col ≈ 41-42) 에 스폰, vx = -3.5. 파동이 바닥을 따라 왼쪽으로 이동.
4. **점프 오버.** Reed 가 파동이 그에게 도달하기 전 점프. 파동이 아래로 지나감; Reed 가 안전하게 착지.
5. **스프린트 패스.** Reed 가 (X-홀드) 파동을 4.9 px/frame 으로 지나서 스프린트; -3.5 px/frame 의 파동이 그를 잡지 못함. Reed 가 오른쪽으로 이동하는 동안 파동이 왼쪽으로 이동, Reed 가 파동을 청소하는지 검증.
6. **보스에 도끼 적중.** Reed 가 `recover` 동안 Warden 의 가슴 시질에 도끼 던짐. 도끼가 히트박스 접촉 → 보스가 hurt 플래시 → `bracken.hp` 감소 → 도끼 디스폰.
7. **도끼 vs. 쇼크웨이브.** Reed 가 활성 쇼크웨이브에 도끼 던짐. 파동 + 도끼 둘 다 디스폰. Reed 가 산다.
8. **보스 킬.** 여섯 도끼 적중이 `recover` 윈도우 동안 깔끔하게 착지. 여섯 번째 적중이 `dead` 상태 트리거. 사망 anim 재생 (60 프레임). 축하 일시정지 (60 프레임). 페이드 투 블랙. Area Cleared 오버레이 나타남.
9. **Reed 사망 (보스 접촉).** Reed 가 Warden 의 몸으로 걸어 들어감. Dying anim + `mile_16` 에서 리스폰 + 보스가 전체 HP 로 리셋 + 카메라 잠금 해제. Reed 가 col 32 로 다시 걸어감; 카메라가 재-잠금; 파이트 재시작.
10. **Reed 사망 (쇼크웨이브).** Reed 가 접촉-프레임에 바닥에 있음. #9 와 같은 흐름.
11. **보스 안에서 GAME OVER.** 아레나에서 3 라이프 잃음. GAME OVER 오버레이 나타남. 아무 키나 누름 → Continue → 전체 Area 리셋 → Reed 가 Stage 1 col 0 에 무장 해제로 스폰.
12. **어떤 비트에서도 콘솔 에러 없음.**

## 15. release-master 를 위한 오픈 질문

1. **보스 난이도 보정.** 나는 HP 6, windup 45 프레임, recover 90 프레임을 제안했다. 첫-패스 플레이테스트가 파이트가 너무 짧다고 (2 공격 사이클로 클리어) 보여주면, HP 를 7 로 올리고 windup → 30 프레임을 고려. 너무 길면, HP 를 5 로 떨어뜨림. Design 이 최종 애니메이션을 출시하기 전 목표 클리어-시간 결정 — windup-프레임 카운트가 애니메이션 페이싱을 구동.

2. **보스 HP 디스플레이.** Reed 가 Warden 의 HP 를 가시적 바 (라이프 아래, 화면-상단 보조 HUD) 를 통해 봐야 할지, 아니면 보스의 실루엣 (HP 가 떨어지면서 시질이 페이드 / 균열 보임) 에서만 읽어야 할지? 권장: v0.75 용 **실루엣 읽기** (더 우아함, 새 HUD 에셋 없음); 플레이테스트가 혼란을 보고하면 나중에 패치에서 명시적 바 추가. 늦은 HUD-아트 작업을 피하기 위해 지금 플래그.

3. **Continue → 보스 룸 단축키.** 현재 Continue 는 Stage 1 에서 전체 Area 를 재시작. 접근성용 대안 고려: **첫** Continue 는 Stage 1 에서 재시작 (의도된 런); 같은 세션의 **후속** Continue 는 Stage 4 스폰에서 시작 ("나는 그냥 보스와 싸우고 싶다" 자비). v0.75 에 권장하지 않음 (범위 밖, 여정을 희석), 하지만 release-master 가 앞으로 당기고 싶다면 플래그.

4. **Area 2 ahead in v0.85.** Area Cleared 오버레이 카피가 v0.85 패치를 주장. `CLAUDE.md` 쿼타일 표 따라, v0.75 는 멀티-에어리어 쿼타일이고 v1.0 은 풀-콘텐츠; v0.85 쿼타일이 없음. 카피는 릴리스-정직 읽기지만 패치 번호가 존재하지 않음. 실제 미래-포인터 선택: **"v1.0 에 더 많은 지형이 있다"**, **"문턱이 다른 길로 열린다 — 지금은"**, 또는 **"길은 이어진다 — 곧"** (v0.50.1 의 Stage-Cleared 부드러운 카피 재사용). 바이링구얼 페어 양쪽이 어쨌든 저자 필요.

5. **보스 오디오.** 본 브리프 범위 밖 (story-lead 가 오디오 명세 안 함). 하지만: Warden 의 `windup` 은 구별되는 빌드-업 사운드 (낮은 럼블) 가 있어야 하고, `attack` 슬램은 타악기적 thud (`docs/story/world.md` 따라 woody/wet) 여야 하고, `dead` 애니메이션은 으르렁이나 비명보다는 세계의 시그니처 "작은 들숨"으로 끝나야 함. design-lead 의 오디오 동반 (브리프된 경우) 에 플래그.
